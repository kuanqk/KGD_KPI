# Материалы КГД: папка `old/` и хронология

Документ описывает **локальный архив** выгрузок от КГД, переписок и справочных файлов. Используется при работе с ETL из витрин КГД и дальнейшем развитии импорта.

## Важно про Git

Каталог **`old/`** указан в `.gitignore` и **не попадает в репозиторий**. У нового разработчика папка может отсутствовать: материалы передаются отдельно (общий диск, архив от коллег). Этот файл в `docs/` фиксирует **структуру и смысл** папок, чтобы не потерять контекст.

## Соглашение по именам

| Шаблон | Значение |
|--------|----------|
| `260406`, `260410` и т.п. | Дата **ГГММДД** (например `260410` = 2026-04-10) |
| `260407-2` | Вторая поставка за тот же период / дополнение в тот же день |
| Вложенные папки (`08.04`) | Дополнительная группировка по дате внутри дня |

Первый пакет от Алихана: `comments.txt`, `una_departments.csv` (whitelist `de_department_id` для УНА), CSV по делам/актам/периодам, XML, скриншоты, `de_tax_payer_audit_reason.txt`, справочные выгрузки.

### `old/260406`

Переписка (`message.txt`, `message2.txt`), скрин `img03.jpeg` — уточнения по БД и процессу.

### `old/260407`

Ответы в `message_1part.txt`, `message_2part.txt`. Подпапка `tax_act/` — XML-вью по актам: `act_kind`, `de_audit_initiator_id`, даты, связи с делом.

### `old/260407-2`

`pay_sums_tables.zip` и распакованная `pay_sums_tables/` — XML по таблицам сумм (`ri_tax_act_accrued_total`, `ri_collected_amount_record`, `ri_tax_act_group_pay_tax` и др.). `msg03.txt` — короткие комментарии.

### `old/260408`

`msg05.txt`. Подпапка `08.04/` — консолидированные вью: `act_all_data_wo_sums`, `act_and_accrued_total`, варианты по пеням и контролю.

### `old/260408-2`

`test.xml` — join акта с `ri_tax_act_accrued_total`, фильтр `act_kind=2`; использовался для подсчёта уникальных актов и строк по КБК.

### `old/260409`

`msg06.txt` — переписка о сверке. `для проверки.xlsx` — ручная сверка `accrued` / `penalty` с листом «Завершенные данные ДГД» (Excel Олжаса).

### `old/260410`

`msg07.txt` — приоритет взыскания и штрафов, отложенные апелляции.  
`act_collect.xml` — связка акта с «предположительными» взысканиями (`amount`, `collection_date` чаще пустые).  
Длинный файл `_select_from_isna_data_act_all_data_wo_sums_v2_...xml` — LEFT JOIN акта, `act_and_accrued_total` и collect на ограниченный список ИИН (`act_kind=2`, `accrued IS NOT NULL`).

### `old/260413`

`msg08.txt` — подтверждение таблицы `ri_collected_amount_record`: мало строк (~200 в ранней выгрузке), таблица новая; ввод взысканий в ИС отделами — **с октября 2025**; сверка по ИИН с Excel Олжаса даёт совпадения; план — уточнить объём в прод-таблице, стимулировать ввод в отделах, повторить проверку.  
`iin_collected_amount_act.xml` — черновой JOIN (ИИН + взыскание + акт).  
`answer01.jpeg` — скрин запроса `isna_data.de_collection_method`: таблица **`de_collection_method`**, 3 строки (`code` 1–3: ПП, переплата, доп. налоговая отчётность); FK из `ri_collected_amount_record.de_collection_method_id`.

### `old/260414`

Три полных справочника XML от Алихана: **`de_audit_form.xml`** (17 форм проверки), **`de_collection_method.xml`** (3 метода взыскания), **`de_audit_initiator.xml`** (40 органов инициирования).  
**`olzhas feedback.jpeg`** — переписка: для KPI 5 исключать проверки по **`de_audit_initiator.code` ∈ { 6, 30, 31, 38, 39, 42, 47 }; список отправлен Олжасу Алиханом.  
**`completed_acts_v2.xml`** — вью завершённых актов без лишних дублей; дубликаты `act_number` только из‑за **нескольких КБК** на акт; суммы по акту в вью **не агрегированы** (для картины по строкам).  
**`для проверки.xlsx`** — сверка с Excel ДГД: жёлтым отмечены акты, найденные **один раз** в Excel; синим — **несколько строк** с разными КБК; зелёные — ранее уже разобранные кейсы. Расхождения по суммам (налог/пеня) — **3 ячейки из 40**; даты, коды КБК и ДГД и пр. — **1:1** с БД; Алихан считает данные **в БД** корректнее Excel в спорных ячейках.

### `old/local_docs`

Локальные копии **не из закрытого контура КГД**: ТЗ, Excel «Статистика КЭР РК на 01.01.2026» и др. Используются как эталон для сверки KPI.

### `old/git_errors_ci`

Артефакты CI (логи прогонов). К маппингу БД КГД не относится.

## Хронология выводов (кратко)

Сводка по состоянию на апрель 2026; детали — в [business/data_sources.md](../business/data_sources.md).

1. **Номера:** TAC — предписание/дело; OAP — акт налоговой проверки (основной номер акта в `ri_tax_act`).
2. **Доначисления:** строки в `ri_tax_act_accrued_total` по одному акту делятся по **КБК** (`de_kbk_id`, в выгрузках колонка `code`); агрегация — `SUM` по акту.
3. **Пеня:** для ряда кейсов совпадает с Excel; отдельные кейсы (разные даты предварительного и окончательного акта) требуют согласования с заказчиком — см. переписку в `msg06.txt`.
4. **Взыскание:** цепочка `ri_collected_amount_record` → `re_tp_reg_card` ← `ri_tax_act_tax_payer` → `ri_tax_act`; таблица подтверждена сверкой с Excel (апр. 2026), но **объём данных ограничен** (ввод с октября 2025). В выгрузке `260413` — ~662 строк, у большинства заполнены `act_id` / `document_number`. См. `msg08.txt`, `iin_collected_amount_act.xml`. Ранее в `act_collect` большинство `amount` были пустые — см. `260410`.
5. **Предмет проверки для дедупликации:** КБК из той же таблицы доначислений + период проверки из акта/дела.
6. **Справочники (апрель 2026):** `de_audit_form`, `de_collection_method`, `de_audit_initiator` — см. `old/260414/*.xml`. KPI 5: исключение по кодам органов инициирования (список Олжаса).
7. **Вью `completed_acts_v2`:** одна строка на связку акт + КБК; повтор номера акта = несколько КБК; агрегация сумм — на стороне ETL (`GROUP BY` акт). Сверка с Excel: **`для проверки.xlsx`** — 3/40 ячеек по суммам (налог/пеня), остальное 1:1.
