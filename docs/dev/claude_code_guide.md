# Работа с Claude Code — экономия токенов

---

## Принцип минимального контекста

Вместо того чтобы давать Claude Code весь проект — давать только нужные файлы.

### Шаблон промпта

```
Прочитай [нужные файлы из docs/].

[Задача]

Не запускай команды. Только создай/измени файлы.
```

---

## Какой контекст давать для каждой задачи

| Задача | Файлы контекста |
|--------|----------------|
| Работа с KPI Engine | `docs/apps/kpi.md` + `docs/business/kpi_formulas.md` |
| Работа с ETL / импортом | `docs/apps/etl.md` + `docs/business/data_sources.md` |
| Работа с пользователями / RLS | `docs/apps/core.md` |
| Работа с экспортом | `docs/apps/reports.md` |
| Работа с API | `docs/architecture/api.md` + `docs/apps/[нужный].md` |
| Работа с Docker | `docs/infrastructure/docker.md` |
| Новый спринт (общий) | `docs/README.md` + `docs/dev/conventions.md` |
| Спринт 18 (реальные данные) | `docs/apps/etl.md` + `docs/business/data_sources.md` |

---

## Что НЕ давать Claude Code

- Все файлы `migrations/`
- Все файлы `tests/` (если не пишем тесты)
- `frontend/` (если задача бэкенд)
- `apps/` целиком (только нужный app)
- `docker/`, `.github/` (если не работаем с инфрой)

---

## Обязательная строка в каждом промпте

```
Не запускай команды. Только создай/измени файлы.
```

Это экономит токены и время — команды (migrate, test) разработчик запускает локально.

## Шаблоны промптов по типу задачи

### Новая фича в backend app

```
Прочитай docs/apps/[app].md и docs/dev/conventions.md.

[Задача]

Не запускай команды. Только создай/измени файлы.
```

### Исправление бага в KPI Engine

```
Прочитай docs/apps/kpi.md и docs/business/kpi_formulas.md.

Баг: [описание]
Ожидаемое поведение: [что должно быть]

Не запускай команды. Только измени файлы.
```

### Новый API эндпоинт

```
Прочитай docs/architecture/api.md и docs/apps/[app].md.

Добавь эндпоинт: [описание]
Права доступа: [роли]
RLS: [нужен / не нужен]

Не запускай команды. Только создай/измени файлы.
```

### Спринт 18 (замена условных полей)

```
Прочитай docs/apps/etl.md и docs/business/data_sources.md.

Получена реальная schema БД КГД. Таблица маппинга:
[вставить таблицу реальных полей]

Обнови normalizer.py и importer.py.
Обнови таблицу условных полей в docs/business/data_sources.md.

Не запускай команды. Только создай/измени файлы.
```

## После каждого спринта

1. Запустить локально: `make migrate`, тесты
2. Коммит: `git add . && git commit -m "Sprint N: ..."  && git push`
3. Обновить статус в `docs/sprints/completed.md`
