# KGD KPI Monitor — Project Context for Claude Code

## Что это
Веб-платформа для автоматического расчёта KPI по 20 региональным департаментам государственных доходов (ДГД) Казахстана. Заменяет ручной Excel-процесс (1–2 дня) на расчёт в реальном времени.

## Стек
- **Backend:** Python 3.12, Django 5.x, Django REST Framework
- **Frontend:** Vue 3 (отдельный SPA, общение через DRF API)
- **БД:** PostgreSQL 16
- **Очереди:** Celery + Redis
- **Инфраструктура:** Docker Compose (web, db, redis, worker, beat, nginx)
- **Экспорт:** openpyxl (XLSX), WeasyPrint (PDF)
- **Карта:** Leaflet.js + GeoJSON (20 регионов Казахстана)

## Источники данных

### Два источника — маршрутизация по дате
- До **09.07.2025** → данные из ИС **ИНИС**
- С **09.07.2025** → данные из ИС **ИСНА**
- Маршрутизация реализуется при ETL-импорте по полю даты записи

### Глобальные фильтры (применяются ко всем KPI 1–3)
- `management = 'УНА'` — только Управление налогового аудита (есть также УКН, УНН, УДМ, УАКН и др.)

### Флаги учёта (поля «да/нет» в исходных таблицах)
| Источник | Поле-флаг | Фильтр | Правила исключения |
|----------|-----------|--------|--------------------|
| Завершённые ИНИС/ИСНА | `is_counted` | `= True` | Проставляет оператор вручную |
| Завершённые данные ДГД | `is_accepted` | `= True` | После сверки с данными КГД |
| Проводимые ИНИС/ИСНА | `is_counted` | `= True` | Исключить: по уголовным делам, по запросам правоохранит. органов |
| Отменённые акты | `is_counted` | `= True` | Исключить: акты >2 лет до решения Апелл. комиссии; акты не-УНА |
| Свод доначислений ДГД | `is_norm` | `= True` | После сверки с данными КГД |

### ДФНО
ДФНО (дополнительные формы налоговой отчётности) — специальный тип проверок.
- KPI 1 (доначисление) → **включать**
- KPI 2 (взыскание) → **включать**
- KPI 3 (среднее на проверку) → **исключать** (`form_type != 'ДФНО'`)

## KPI 1 — Доначисление (макс. 10 баллов)

```
Факт        = SUM(amount_assessed) из ИНИС/ИСНА
              WHERE management='УНА' AND period IN [date_from, date_to]

KPI_план    = kbk_share_pct[dgd] / 100 * SUM(KPI_план всех ДГД прошлого года) * 1.20
              (kbk_share_pct — ручной ввод оператором 1 раз в год)

Исполнение  = Факт / KPI_план

Баллы:
  ≥ 100%  → 10
  ≥ 90%   → 5
  < 90%   → 0
```

## KPI 2 — Взыскание (макс. 40 баллов)

```
Факт        = SUM(amount_collected) из ИНИС/ИСНА
              WHERE management='УНА' AND is_accepted=True AND period IN [date_from, date_to]

KPI_план    = kbk_share_pct[dgd] / 100 * SUM(KPI_план взысканий всех ДГД прошлого года) * 1.20

Исполнение  = Факт / KPI_план

Баллы:
  ≥ 100%  → 40
  ≥ 90%   → 20
  ≥ 80%   → 10
  < 80%   → 0
```

## KPI 3 — Среднее доначисление на 1 проверку (макс. 10 баллов)

```
Сумма       = SUM(amount_assessed) из ИНИС/ИСНА
              WHERE management='УНА' AND is_counted=True AND form_type != 'ДФНО'

Кол-во      = COUNT(inspections) из ИНИС/ИСНА
              WHERE management='УНА' AND is_counted=True AND form_type != 'ДФНО'

Среднее_факт = Сумма / Кол-во

KPI_план    = (Среднее по всем 20 ДГД за прошлый год) * 1.20
              ЕДИНЫЙ порог для всех ДГД (не индивидуальный)

Исполнение  = Среднее_факт / KPI_план

Баллы:
  ≥ 100%  → 10
  ≥ 90%   → 5
  < 90%   → 0   ← диапазон 80–89% тоже даёт 0 (не 5)
```

## KPI 4 — Коэффициент занятости (макс. 15 баллов)

```
Проверок    = COUNT(completed_inspections) из ИНИС/ИСНА
              WHERE management='УНА' AND is_counted=True

Сотрудников = staff_count[dgd]  ← ручной ввод оператором 1 раз в год

Месяцев     = кол-во полных месяцев с 01.01 по дату отчёта

Коэф        = Проверок / Сотрудников / Месяцев

Баллы:
  ≥ 0.5   → 15
  ≥ 0.4   → 5
  < 0.4   → 0
```

## KPI 5 — Доля проверок > 6 месяцев (макс. 10 баллов)

```
Все_проводимые = COUNT(active_inspections) из ИНИС/ИСНА
                 WHERE is_counted=True
                 ИСКЛЮЧИТЬ: уголовные дела, запросы правоохранит. органов

Долгие         = COUNT(active_inspections)
                 WHERE is_counted=True
                   AND (report_date - prescription_date) > 180 дней

Доля           = Долгие / Все_проводимые * 100%

Баллы:
  < 20%   → 10
  ≥ 20%   → 0
```

## KPI 6 — Удельный вес отменённых сумм (макс. 15 баллов)

```
Источник       = отдельная таблица обжалований (ИНИС)
                 WHERE is_counted=True
                 ИСКЛЮЧИТЬ: акты завершённые >2 лет до решения Апелл. комиссии
                 ИСКЛЮЧИТЬ: акты управлений кроме УНА

Отменено       = SUM(amount_cancelled)
Доначислено    = Факт из KPI 1 (для того же ДГД и периода)

Доля           = Отменено / Доначислено * 100%

Баллы:
  ≤ 1%    → 15
  ≤ 2%    → 5
  > 2%    → 0
```

## Роли пользователей

| Роль | Код | Что может |
|------|-----|-----------|
| Администратор | `admin` | CRUD пользователей, назначение ролей, просмотр всех логов, мониторинг ETL |
| Оператор | `operator` | Загрузка данных, управление формулами, запуск расчётов, ручной ввод (доли КБК, штат), пометка аномалий, отправка на утверждение |
| Проверяющий | `reviewer` | Утверждение/возврат отчётов с комментарием, запрос пересчёта |
| Наблюдатель | `viewer` | Только чтение утверждённых отчётов — только по своим регионам (RLS) |

### RLS (Row-Level Security)
- Наблюдатель видит данные только по регионам, к которым привязан (`UserRegion`)
- Реализовать через кастомный QuerySet-миксин `RegionScopedQuerySet`
- Все view наследуют миксин, кроме Администратора

## Периоды расчёта
- **Основной режим:** накопленный с 01.01 по дату отчёта
- **Произвольный период:** поддерживается (`date_from` / `date_to`)
- **Каскад:** за июнь = январь–июнь (накопленный итог)

---

## Версионирование формул
- Формулы хранятся в модели `KPIFormula` (JSON-конфиг + версия)
- При изменении формулы → создаётся новая версия, старые результаты не пересчитываются
- Каждый `KPIResult` привязан к конкретной версии формулы

---

## Аудит-лог (сквозное требование)
Все действия логируются в `AuditLog`:
- Импорт данных (источник, кол-во записей, статус)
- Изменение формул (старая/новая версия, автор)
- Расчёт KPI (период, инициатор, версия формулы)
- Входы/выходы (IP, MAC-адрес)
- Экспорт отчётов
- Ручные корректировки
- Управление учётками

Срок хранения: бессрочно. Просмотр только для Администратора.
